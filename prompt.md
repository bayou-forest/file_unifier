# file_unifier プログラム概要

以下のプログラムを作成し、概要をreadme.mdに記述してください。一般的な開発プロセスに沿って実施してください。

## 目的・概要

写真などのバックアップや整理のため、2つのフォルダを比較し、重複ファイルを確認、一意に集約するためのアプリケーションです。

## 要件

- Windwos/Mac両方で動作するアプリケーションとしたい
- デスクトップアプリケーションにしたい。Windwosはインストーラを作りたい。
- フォルダを2つ選ぶ
  - フォルダに含まれるファイルを再帰的にすべて網羅的に調べる。ファイルのファイルパス、ファイル名、md5ハッシュ値をチェックする。
  - 1つのルートフォルダ配下ファイル群のなかでハッシュが重複しているファイルを一覧にする
  - 選択した2つのフォルダ配下ファイル間でハッシュが重複しているファイルを一覧にする
  - 1つのルートフォルダ配下ファイル群のなかでファイル名が重複しているファイルを一覧にする
  - 選択した2つのフォルダ配下ファイル間でファイル名が重複しているファイルを一覧にする
  - それぞれの一覧に対して、「片側を一括削除」、「ダイアログで1つずつどちらを残すか決める」ができるようにする。
- サブ機能として、ファイルのフラット化機能を用意する。ファイルのフラット化は、特定のフォルダパスを選び、そのフォルダの子階層以下のファイルをフォルダパス配下に移動させる機能である。ファイル名が重複して移動ができない場合は、そのまま残し、移動できなかったファイルパスを表示する。ファイルの処理順は任意でよい。
- サブ機能として、特定のルートフォルダ配下の、空のフォルダを再帰的に全削除する機能を用意する。

4. スキャンの中断ボタンを用意してください。中断した場合、計算済みのファイルパスとハッシュ値はキャッシュされ、途中から高速に再開できるようにしてください。キャッシュした値はアプリの再起動でクリアされるようにしてください。
